# AI Agent 游戏开发工作台操作指令集

**版本：1.0**
**适用对象：Claude 3.5 Sonnet, GPT-4o, 及其他具备代码/文件操作能力的 AI 助手**

本指令集旨在指导 AI Agent 接管并自动化游戏开发工作台中的核心任务。通过本手册，AI 将学会如何“导入”外部 AI 模块（如 Leonardo.ai），并按照预设的工作流完成从策划、美术到代码的闭环开发。

---

## 1. 模块导入与环境配置指令

在开始工作前，请先确保已“导入”以下核心能力模块。

### 1.1 模块 A：Leonardo.ai (美术资产生成)
*   **功能**：生成高质量角色立绘、装备图标、UI 元素。
*   **接入方式**：REST API 调用。
*   **开发者指令**：
    > “请使用 `Leonardo.ai API` 生成美术资产。你需要调用 `https://cloud.leonardo.ai/api/rest/v1/generations` 接口。请确保生成的 Prompt 符合‘黑暗性感西幻’风格，并根据需要配置 `modelId` 和 `controlNet` 参数。生成完成后，请下载并保存图像到项目指定目录。”

### 1.2 模块 B：Unity/C# 核心库 (代码逻辑实现)
*   **功能**：编写、重构和调试游戏逻辑。
*   **接入方式**：本地文件系统读写。
*   **开发者指令**：
    > “你现在是一名资深 Unity 开发者。请直接输出符合 C# 规范的代码块，并明确标注文件名和路径。输出格式必须为 Markdown 代码块，以便我解析并自动写入 `/Assets/Scripts/` 目录。”

### 1.3 模块 C：数值平衡分析仪 (数据处理)
*   **功能**：分析模拟数据，优化数值表。
*   **接入方式**：CSV/JSON 文件解析。
*   **开发者指令**：
    > “请读取 `/ProjectData/Simulation/` 目录下的 CSV 数据文件。分析各职业的收益曲线和死亡率，并以 Markdown 表格形式提供数值调整建议。”

## 2. 核心自动化工作流指令

### 2.1 工作流一：玩法系统“从设计到代码”全自动实现
**适用场景**：新增一个技能系统、装备系统或挂机逻辑。

**Agent 执行步骤**：
1.  **需求解析**：阅读用户提供的玩法描述。
2.  **GDD 生成**：输出 Markdown 格式的系统设计文档。
3.  **架构设计**：输出类图（Mermaid）和伪代码。
4.  **代码实现**：输出完整的 C# 代码块。
5.  **自动化指令示例**：
    > “请根据用户描述的‘暗影法师召唤系统’，先撰写设计文档，然后直接输出对应的 `ShadowSummoner.cs` 和 `SummonedMinion.cs` 完整代码。确保包含注释，并符合 Unity 的 `MonoBehaviour` 规范。”

### 2.2 工作流二：美术资产“从需求到部署”全自动生成
**适用场景**：批量生成 50 个黑暗风格的戒指图标。

**Agent 执行步骤**：
1.  **Prompt 优化**：将简单的需求转化为 Leonardo.ai 专用 Prompt。
2.  **API 调用**：发起批量生成请求。
3.  **结果轮询**：获取图像 URL。
4.  **资产部署**：下载图像并命名为 `Icon_Ring_01.png` 等，保存至 `/Assets/Art/UI/Icons/`。
5.  **自动化指令示例**：
    > “请调用 Leonardo.ai API 批量生成 10 个‘被诅咒的黑暗戒指’图标。风格要求：哥特、性感、暗红光芒、精致。生成后请自动重命名并保存到项目图标文件夹。”

### 2.3 工作流三：数值平衡“从模拟到优化”闭环调整
**适用场景**：调整战士和法师的挂机平衡。

**Agent 执行步骤**：
1.  **数据读取**：解析最近 1000 次模拟挂机的日志。
2.  **异常识别**：寻找收益过高或死亡过频的临界点。
3.  **表单更新**：直接输出更新后的 `GameBalance.json` 或配置代码。
4.  **自动化指令示例**：
    > “请分析最近的战斗日志，如果法师在 10-20 级死亡率超过 20%，请直接生成一份调整后的 `PlayerStats.cs` 基础属性代码，适当提高法师的基础血量和护甲。”

## 3. 开发者与 Agent 协作协议 (Best Practices)

1.  **Markdown 优先**：所有非代码输出均使用 Markdown 格式，以便于阅读和自动解析。
2.  **文件路径明确**：Agent 在输出代码时，必须在代码块上方标注绝对路径或项目相对路径。
3.  **版本注释**：在自动生成的代码或文档中，包含 `// AI Generated - [Date]` 标签，方便追溯。
4.  **分步确认**：对于涉及核心架构的大规模变动，Agent 应先输出“修改方案”供开发者确认，再执行具体代码写入。
