# Coop Repo

This directory is the handoff channel for agents working on this repository.

## Reading Order

1. Read `LATEST.md`.
2. Read the report linked by `LATEST.md`.
3. Follow links to detailed design documents or source files only when relevant.
4. Inspect `git status` before making changes.

## Writing Rules

- One completed work unit produces one new Markdown report in `reports/`.
- Use filenames in the form `YYYY-MM-DD_HHMM_short-topic.md`.
- Reports are append-only history. Never replace or rewrite an older agent's report.
- Copy `REPORT_TEMPLATE.md` for the report structure.
- Update `LATEST.md` only after the work and its validation are complete.
- Clearly distinguish completed work, unverified work, and recommended next work.

`LATEST.md` is the fast entry point. The reports directory is the durable history.
