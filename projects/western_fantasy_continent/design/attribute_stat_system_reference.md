# 大属性 / 小属性数值体系参考

本文档记录角色养成与装备系统的属性框架参考。它借鉴 `Dark and Darker` 的大属性驱动小属性结构，但目标是服务本项目的自走战斗、角色搭配、刷装养成，而不是照搬动作游戏数值。

## 核心定义

### 大属性

大属性是玩家最容易理解的成长语言。它通常出现在角色升级、职业基础、装备高价值词条上。

大属性不直接等于最终战斗表现，而是同时推动多个小属性。

示例：

- 力量：攻击、少量生命、物理压制
- 体魄：生命、防御、受治疗
- 敏捷：攻速、移动、闪避
- 技巧：暴击、命中、标记/处决收益
- 意志：魔抗、治疗、护盾
- 奥秘：法强、技能急速、元素伤害
- 统御：团队增益、召唤/光环、控制强度

### 小属性

小属性是战斗公式真正读取的结算语言。它们更窄、更明确，适合装备部位、流派专精、机制词条。

示例：

- 攻击
- 生命
- 防御
- 魔抗
- 攻速
- 暴击率
- 暴击伤害
- 法强
- 技能急速
- 治疗强度
- 护盾强度
- 火焰伤害
- 剧毒伤害
- 雷电伤害
- 冰霜伤害
- 元素增幅
- 持续时间增幅
- 穿透
- 抗性

## 这个系统解决什么问题

### 1. 防止角色极端化

这是大属性体系最关键的优势之一。

如果装备只给小属性，玩家很容易堆出极端角色，比如：

- `100 攻击 / 5 生命`
- `0 输出 / 无限护盾`
- `极限攻速 / 没有任何基础伤害`

这种角色可能在某些对局里非常离谱，也可能在另一些对局里完全无效。它们会让平衡变得很脆，玩家理解成本也很高。

大属性同时提升多个相关小属性，可以天然把角色拉回合理形状。

例如：

- 力量不只加攻击，也给一点生命或物理压制。
- 体魄不只加生命，也给防御或受治疗。
- 奥秘不只加法强，也给技能急速或元素收益。

这样玩家仍然可以专精，但专精不会变成失控的单点堆叠。

### 2. 装备更容易读

`+3 力量` 比 `+6 攻击、+18 生命、+2% 物理压制` 更容易被玩家快速理解。

大属性适合做装备的高级表达，小属性适合做详细展开。

推荐 UI 表达：

- 装备格子：显示 `+3 力量`
- 详情面板：展开显示 `攻击 +6 / 生命 +18 / 物理压制 +2%`

### 3. 职业定位更自然

职业可以通过基础大属性形成天然倾向。

例如：

- 战士：力量、体魄
- 狂战士：力量、敏捷、体魄
- 骑士：体魄、意志
- 游侠：敏捷、技巧
- 法师：奥秘、意志
- 牧师：意志、体魄
- 术士：奥秘、体魄
- 吟游诗人：统御、敏捷

这样装备选择会自然分化，而不是所有职业都只看“战力更高”。

### 4. 给装备稀有度提供层级

低级装备可以主要给小属性，直接、明确。

高级装备可以给大属性，提供复合收益。

更高级装备再加入机制词条。

层级示例：

- 普通：`攻击 +8`
- 稀有：`攻击 +10`，或 `力量 +1`
- 史诗：`力量 +2 / 暴击率 +3%`
- 传说：`力量 +3 / 攻速 +4% / 对前排伤害 +5%`
- 神话：`力量 +4 / 攻速 +6% / 冲锋后破盾 +10%`

## 优势

### 复合成长

一个词条能带来多个小收益，装备会更有质感。

### 降低极端 build 风险

大属性会把角色拉成一个相对完整的战斗单位，避免单项数值堆到畸形。

### 更容易做职业幻想

玩家看到某个职业吃什么大属性，就能理解它的成长方向。

### 更容易做边际递减

可以在大属性转小属性时设置收益曲线。前期收益明显，后期逐渐变慢。

### 更容易控制装备上限

装备可以给大属性，但大属性会分散成多个小属性，不会像直接给 `+100 攻击` 那样瞬间击穿平衡。

## 劣势

### 不够直观

玩家看到 `+2 意志`，可能不知道到底加了多少治疗、魔抗、护盾。

解决方式：详情面板必须展开派生结果。

### 平衡更复杂

改一个大属性，可能同时影响输出、生存、治疗、控制。

解决方式：大属性数值要集中配置，并配套模拟矩阵。

### 容易出现万能属性

如果某个大属性同时加输出、生存、资源效率，而且数值都高，它会变成无脑最优。

解决方式：

- 每个大属性必须有主要功能和次要功能。
- 次要功能收益要明显低于主要功能。
- 不允许一个大属性覆盖所有战斗需求。

### 小属性可能失去存在感

如果大属性太强，小属性词条就没人看。

解决方式：

- 大属性负责“完整性”。
- 小属性负责“专精”。
- 机制词条负责“流派变化”。

## 推荐结构

### 大属性层

```text
力量 / 体魄 / 敏捷 / 技巧 / 意志 / 奥秘 / 统御
```

### 小属性层

```text
基础战斗：
生命、攻击、防御、魔抗、攻速、移动、暴击、暴伤、法强、技能急速

资源/功能：
治疗强度、护盾强度、受治疗、控制强度、控制抗性

元素与流派：
火焰伤害、剧毒伤害、雷电伤害、冰霜伤害、神圣伤害、暗影伤害、奥术伤害
元素增幅、持续时间、穿透、抗性

形态：
单体伤害、范围伤害、持续伤害、普攻伤害、技能伤害、反击伤害、召唤伤害
```

## 推荐映射草案

这些数值不是最终平衡，只是第一版关系草案。

| 大属性 | 主要提升 | 次要提升 | 设计目的 |
| --- | --- | --- | --- |
| 力量 | 攻击、物理伤害 | 少量生命 | 物理输出不至于太脆 |
| 体魄 | 生命、防御 | 受治疗 | 前排和生存底座 |
| 敏捷 | 攻速、移动 | 闪避 | 频率流和机动流 |
| 技巧 | 暴击、命中 | 标记/处决收益 | 精准单点和爆发 |
| 意志 | 魔抗、治疗、护盾 | 控制抗性 | 支援和抗魔 |
| 奥秘 | 法强、技能急速 | 元素伤害 | 法术与机制输出 |
| 统御 | 团队增益、光环 | 召唤/控制强度 | 团队流派和辅助核心 |

## 装备设计原则

### 部位固定倾向

每个装备部位应该有固定的大属性/小属性倾向，避免全身都能 roll 所有东西。

示例：

- 武器：攻击、法强、元素伤害、暴击
- 头盔：技巧、意志、暴击、技能急速
- 胸甲：体魄、防御、魔抗、生命
- 手套：力量、敏捷、攻速、暴击
- 腿甲：体魄、敏捷、防御、控制抗性
- 鞋子：敏捷、移动、闪避、技能急速
- 戒指：大属性、元素增幅、暴击、机制词条
- 项链：大属性、治疗、护盾、技能急速

### 稀有度决定词条数量

参考结构：

- 普通：1 个基础小属性
- 稀有：1 个较强小属性，或 1 个弱大属性
- 史诗：2 个词条
- 传说：3 个词条
- 神话：3 个词条 + 机制效果

### 大属性和小属性的关系

大属性提供完整性，小属性提供专精。

例子：

- `力量 +3`：攻击提升，同时补一点生命。
- `攻击 +18`：更纯粹，输出更尖锐，但不补生存。

两者都应该有价值。
