# 数值统一方案 v2 候选

本文档记录当前对齐后的属性体系候选。

状态：设计候选，未实装，未完成数学模拟验收。

目标：把角色成长、装备词条、技能收益统一到一套可计算的属性结构里，同时保持刷宝游戏的直观成长感。

## 一、核心原则

### 1. 先简单，后细分

早期玩家主要理解这些数值：

- 攻击
- 法强
- 生命
- 防御
- 攻速
- 技能急速

中后期再逐步引入：

- 效果抗性
- 效果强度
- 暴击
- 吸血
- 真伤
- DOT 增幅
- 控制强度

这样可以避免一开始就把系统做成一堆难懂词条。

### 2. 单发伤害和效果伤害分开

普通单发伤害走防御。

效果类内容走效果抗性。

```text
单发伤害：普攻、物理技能、法术技能
效果内容：DOT、控制、诅咒、减速、燃烧、剧毒
```

这让 DOT 有独立身份：

- 火球是法术单发，吃法强，被防御减免。
- 毒是效果伤害，主要吃效果强度/DOT 增幅，被效果抗性减免。
- 控制不是伤害，但也是效果，被效果抗性缩短。

### 3. 大属性只做一主一副

每个大属性只允许：

- 一个主收益
- 一个副产物

不要一个属性同时加太多东西。

这样更方便测试，也更容易让玩家理解。

## 二、基础小属性

这些是战斗公式真正使用的核心属性。

| 小属性 | 定位 | 主要作用 |
| --- | --- | --- |
| 攻击 | 物理输出 | 普攻、物理技能、部分斩击/射击技能 |
| 法强 | 法术与功能强度 | 法术单发、治疗、护盾、部分效果基础值 |
| 生命 | 基础生存 | 最大生命值 |
| 防御 | 单发承伤 | 减少所有单发伤害，包括物理单发和法术单发 |
| 攻速 | 普攻频率 | 普攻流、on-hit、吸血、真伤触发等 |
| 技能急速 | 技能循环 | 缩短技能冷却，服务技能流、治疗流、控制流 |

### 防御的定位

防御不区分物理防御和魔法防御。

当前候选逻辑：

```text
所有单发伤害都被防御减免。
DOT、控制、诅咒等效果不被防御减免。
```

这样可以让早期装备成长更简单：

- 攻击/法强让你打得更疼。
- 生命/防御让你更能扛。

## 三、二级属性

二级属性用于中后期 build、装备词条、职业特化。

| 二级属性 | 定位 | 备注 |
| --- | --- | --- |
| 效果抗性 | 抵抗 DOT、控制、诅咒、减速等 | 类似韧性，上限收敛，不能让效果完全失效 |
| 效果强度 | 提高 DOT、控制、诅咒等效果 | 术士、炼金、控制流核心 |
| 治疗强度 | 提高主动治疗 | 牧师、部分诗人 |
| 护盾强度 | 提高主动护盾 | 牧师、骑士、部分诗人 |
| 受治愈增幅 | 自己受到治疗和护盾更多 | 前排、狂战、骑士、前排牧师 |
| 暴击率 | 提高暴击概率 | 游侠、刺客、战士部分流派 |
| 暴击伤害 | 提高暴击收益 | 高风险爆发词条 |
| 吸血 | 造成伤害后回血 | 狂战、战士少量，和攻速联动风险高 |
| 真伤 | 绕过防御和护盾直接扣血 | 高稀有破局属性 |
| DOT 增幅 | 提高持续伤害 | 术士、炼金、燃烧法师 |
| 控制强度 | 提高控制时长或控制效果 | 骑士、冰法、炼金、诗人 |

### 效果抗性上限

效果抗性是万用二级防御，所以必须收敛。

候选规则：

```text
效果抗性最多抵掉 50% 效果。
剩余效果需要靠专门机制处理，例如净化、毒抗、火抗、控制免疫窗口。
```

这可以避免一个属性废掉所有 DOT/控制/诅咒玩法。

## 四、伤害与效果结算

### 单发伤害

```text
最终单发伤害 = 原始单发伤害 * 防御减免
```

单发伤害包括：

- 普攻
- 物理技能
- 法术技能
- 单段爆发
- 单段 AOE

### DOT 伤害

```text
最终 DOT 每跳 = 原始 DOT 每跳 * 效果抗性减免
```

DOT 的成长来源不应该主要依赖法强。

候选方向：

```text
原始 DOT 每跳 = 技能基础值 + 法强 * 低倍率
DOT 最终成长 = 原始 DOT 每跳 * DOT 增幅 * 效果强度修正
```

这让法强能提供一点基础帮助，但 DOT 流真正变强要靠 DOT 增幅、效果强度、层数、持续时间、引爆等机制。

### 控制效果

```text
最终控制时长 = 原始控制时长 * 控制强度修正 * 效果抗性减免
```

控制也属于效果。

### 真伤

候选定义：

```text
真伤不吃防御。
真伤不吃护盾。
真伤直接扣生命。
```

真伤属于高风险、高稀有属性，不能作为常见成长。

## 五、六大属性候选

当前候选大属性如下。

| 大属性 | 主收益 | 副产物 | 定位 |
| --- | --- | --- | --- |
| 武力 | 攻击 | 生命 | 物理输出，带一点近战承载 |
| 坚韧 | 生命 | 防御 | 基础前排、生存成长 |
| 敏捷 | 攻速 | 效果抗性 | 普攻频率、灵活、抗效果 |
| 奥术 | 法强 | 技能急速 | 法术、治疗、护盾强度，带少量循环 |
| 节律 | 技能急速 | 效果强度 | 技能循环、DOT/控制/诅咒流 |
| 韧性 | 防御 | 效果抗性 | 抗压、反 DOT、反控制 |

### 武力

主收益：攻击。

副产物：生命。

设计目的：

- 服务战士、游侠、刺客、部分骑士。
- 物理输出角色不会因为全堆攻击而过脆。
- 狂战虽然主吃敏捷，但副吃武力也能获得一点生命。

### 坚韧

主收益：生命。

副产物：防御。

设计目的：

- 服务传统前排。
- 提供最直观的生存成长。
- 和韧性区分：坚韧是血量主导，韧性是减伤/抗效果主导。

### 敏捷

主收益：攻速。

副产物：效果抗性。

设计目的：

- 服务狂战、刺客、游侠等高频普攻角色。
- 副产物表现为更灵活、更不容易被 DOT/控制拖死。
- 狂战没有直接从敏捷获得生命，这符合当前“狂砍、吸血、但不算高血量”的幻想。

### 奥术

主收益：法强。

副产物：技能急速。

设计目的：

- 服务法师、牧师、术士、炼金、部分诗人。
- 法强统一支持法术单发、治疗、护盾、部分 DOT 基础值。
- 技能急速作为副产物，避免法系属性过窄。

### 节律

主收益：技能急速。

副产物：效果强度。

设计目的：

- 服务技能循环、DOT、控制、诅咒、节奏流。
- 技能急速本身很泛用，因此副产物必须限制它的满收益人群。
- 没有 DOT/控制/诅咒的角色点节律也有用，但会浪费副产物。

### 韧性

主收益：防御。

副产物：效果抗性。

设计目的：

- 服务抗压、反 DOT、反控制。
- 是中后期对抗效果流的主要大属性。
- 不提供输出，所以不会和武力/奥术重叠。

## 六、职业主副属性候选

| 职业 | 主属性 | 副属性 | 理由 |
| --- | --- | --- | --- |
| 战士 | 武力 | 坚韧 | 物理压制 + 前排承载 |
| 狂战士 | 敏捷 | 武力 | 普攻频率 + 物理输出；武力副产物补少量生命 |
| 骑士 | 坚韧 | 韧性 | 生命前排 + 抗压反效果 |
| 游侠 | 武力 | 敏捷 | 远程物理 + 攻速/灵活 |
| 法师 | 奥术 | 节律 | 法强爆发 + 技能循环/效果 |
| 牧师 | 奥术 | 韧性 | 治疗护盾量 + 自保抗压 |
| 术士 | 奥术 | 节律 | 法强基础 + DOT/诅咒/献祭循环 |
| 诗人 | 节律 | 奥术 | 技能循环 + buff/功能强度 |
| 刺客 | 敏捷 | 武力 | 高频切入 + 物理爆发 |
| 炼金师 | 节律 | 奥术 | 效果铺场/循环 + 法强基础 |

## 七、装备词条分层

### 常见词条

适合低中稀有装备大量出现：

- 攻击
- 法强
- 生命
- 防御

### 中高价值词条

出现频率应低于基础词条：

- 攻速
- 技能急速
- 效果抗性
- 效果强度
- 治疗强度
- 护盾强度

### 高风险词条

应放在高稀有装备、套装、藏品、职业专属中：

- 真伤
- 吸血
- 暴击伤害
- 击杀刷新
- DOT 层数上限
- 控制免疫
- 自损转伤害
- 自损转护盾

## 八、当前未决问题

### 1. 防御曲线

防御目前只确定为“减少单发伤害”。

还没有确定具体公式。

候选方向：

- 简单 rating 曲线。
- 类似 Dark and Darker 的分段曲线。
- 按等级区间调整常数的成长曲线。

### 2. 效果抗性曲线

效果抗性倾向于百分比，并设置上限。

未定：

- 每点效果抗性换多少百分比。
- 上限是否固定为 50%。
- 是否对 DOT、控制、诅咒使用同一套衰减。

### 3. DOT 与法强关系

当前候选：

- DOT 可以吃少量法强。
- DOT 主要吃 DOT 增幅、效果强度、层数、持续时间。

未定：

- 法强倍率应该多低。
- 效果强度是否同时影响 DOT 和控制。
- DOT 增幅是否作为独立装备词条保留。

### 4. 狂战生命问题

狂战主属性敏捷不直接给生命。

当前解释：

- 狂战幻想是高频、吸血、低血反打。
- 生命来自副属性武力的副产物，以及装备/吸血/低血机制。

后续需要通过数学模拟验证：狂战是否会因为主敏捷而过脆。

## 九、下一步验证

下一步不是直接实装，而是做数学模拟。

建议模拟问题：

1. 每个职业按主属性、副属性、混点成长，是否有 2-3 条有效路线。
2. 纯攻击、纯法强、纯生命、纯防御是否会形成单一最优。
3. 攻速和技能急速是否互相稀释，还是都变成必点。
4. 效果抗性是否能克制 DOT/控制，但不会让效果流失效。
5. 奥术和节律是否区分明显。
6. 狂战主敏捷、副武力的生存是否合理。
7. 术士/炼金是否能通过节律和效果强度形成独立路线。
8. 牧师吃奥术和韧性是否合理，是否需要额外治疗/护盾词条支持。

## 十、结论

当前候选可以作为数值统一方案 v2。

它的核心结构是：

```text
基础成长：攻击 / 法强 / 生命 / 防御 / 攻速 / 技能急速
效果防御：效果抗性
大属性：武力 / 坚韧 / 敏捷 / 奥术 / 节律 / 韧性
规则：单发伤害吃防御，效果内容吃效果抗性
```

这版比 v1 更适合刷宝挂机游戏：

- 前期属性直观。
- 中期能分普攻流和技能流。
- 后期通过效果抗性、效果强度、DOT、真伤、吸血等词条做 build 深度。
- 每个大属性只有一主一副，便于自动测试和后续调参。
