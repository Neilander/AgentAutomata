# 装备词条自动迭代循环 v1

状态：流程设计，未实装。

目标：把装备系统变成 agent 可以自动开发、自动诊断、自动调整的循环，而不是依赖人工逐条审查装备词条。

## 1. 核心原则

装备系统必须全局统一，像 skill asset 一样被多个页面、模拟器和战斗验证复用。

后续不要在不同页面各自写一套装备规则。装备相关内容应沉淀为统一 asset：

- 装备位
- 装备类型
- 固定属性规则
- 词条注册表
- 部位词条池
- 稀有度规则
- 词条等级曲线
- 掉落生成器
- 自动配装评分器

每轮循环中，只允许修改装备系统规则，例如：

- 部位固定属性
- 词条允许部位
- 词条数值曲线
- 词条稀有度门槛
- 稀有度产出概率
- 同类词条共存限制
- 装备类型和职业倾向

不要在装备循环里直接改技能、职业基础数值、战斗引擎或水表队伍。否则无法归因。

## 2. 一轮循环的输入

每轮循环输入一版装备规则：

1. 装备位规则
   - 头、胸、护手、腿、鞋、挂饰、左手、右手、双手。
   - 双手武器占用左手和右手。

2. 固定属性规则
   - 每个部位有基础倾向。
   - 同一个部位可以有不同装备类型，例如法师手套和战士手套固定属性不同。
   - 固定属性随装备等级、稀有度和装备类型变化。

3. 词条规则
   - 大属性：武力、坚韧、敏捷、奥术、节律、韧性。
   - 一级小词条：攻击、法强、生命、防御、攻速、技能急速、效果抗性。
   - 二级小词条：效果强度、治疗强度、护盾强度、受治愈增幅、暴击率、暴击伤害、吸血、DOT 增幅、控制强度、破盾、破甲、先手等。
   - 三级流派词条：火焰增幅、剧毒增幅、标记强度、隐身持续、处决伤害、低血伤害、低血受治愈增幅、反击伤害、净化效率、光环强度等。

4. 稀有度规则
   - v1 到史诗 4 词条即可。
   - 更高稀有度先预留。
   - 稀有度影响固定属性、词条数量、词条等级上限。

5. 词条等级曲线
   - 初始候选：I=1, II=2, III=4, IV=7, V=12。
   - 这条曲线可以在循环里被调整。

## 3. 模拟玩家刷取

先不真实打怪，直接模拟掉落。

基础掉落模型：

```text
一次刷怪 = 掉落 5 件装备
一次模拟 = 连续刷若干次
```

需要测试不同刷取进度：

| 阶段 | 掉落件数 | 目的 |
| --- | ---: | --- |
| 新手早期 | 10-20 | 是否很快获得可玩提升 |
| 初步成型 | 30-50 | 是否能出现可识别 build |
| 稳定刷装 | 80-120 | 是否进入追求高质量词条阶段 |
| 深刷压力 | 200+ | 是否出现极端堆叠或毕业过快 |

掉落模拟要记录：

- 每件装备的部位、类型、稀有度、固定属性、词条、词条等级。
- 每个词条的出现次数。
- 每个部位的平均价值。
- 每个职业/流派可用装备数量。

## 4. 自动配装

掉落后，agent 需要为角色或队伍自动选择装备。

每个职业/流派需要一个偏好函数。例如：

| 流派 | 偏好 |
| --- | --- |
| 低血狂战 | 敏捷、攻速、吸血、低血伤害、低血受治愈增幅、少量生命/防御 |
| 火法燃烧 | 奥术、技能急速、火焰增幅、燃烧持续、DOT 增幅、效果强度 |
| 毒队 | 奥术/节律、剧毒增幅、剧毒持续、DOT 增幅、效果强度 |
| 暗影刺客 | 敏捷、先手、隐身持续、处决伤害、单体伤害、标记强度 |
| 铁壁骑士 | 坚韧、韧性、防御、护盾强度、反击伤害、嘲讽减伤 |
| 牧师续航 | 奥术、韧性、治疗强度、护盾强度、净化效率、技能急速 |

自动配装要输出：

- 当前背包能配出的最好 build。
- 随机穿搭/半随机穿搭/目标 build 的差距。
- 形成该 build 需要多少掉落件数。
- 缺口词条是什么。

## 5. 四个核心评价函数

### 5.1 Build Diversity：多样性

问题：装备组合之间是否有优劣和选择空间。

计算方向：

```text
同一角色/流派下，生成 N 套候选配装
计算每套配装的静态评分和战斗评分
比较 top 20%、median、bottom 20%
```

理想结果：

- 好 build 明显强于乱穿。
- 中等 build 也能玩。
- 不是所有装备都一样。
- 不是只有一个唯一答案。

异常：

- 分布太窄：刷装没有意义。
- 分布太宽：没刷到关键词条就不能玩。
- top build 全部来自同一个部位或同一个词条：部位/词条过强。

### 5.2 Fantasy Amplification：流派增强

问题：装备是否增强特定幻想，而不是只提高总战力。

每个流派有自己的曲线指标。

低血狂战：

- 低血时间
- 低血后攻击频率
- 吸血总量
- 血量是否从暴毙变成低血波动

火法燃烧：

- 燃烧覆盖率
- 燃烧伤害占比
- 扩散/引爆次数
- AOE 峰值窗口

毒队：

- 毒层平均值
- 毒层持续时间
- DOT 伤害占比
- 对手是否仍有反制窗口

暗影刺客：

- 开场切入成功率
- 锁定目标死亡时间
- 隐身/切入期间承伤
- 是否破坏对方 combo

牧师/骑士：

- 护盾/治疗是否改变死亡时间
- 危机后是否回升
- 是否只是拖长战斗而没有兑现

### 5.3 Extreme Risk：极端化风险

问题：是否出现单一维度堆穿系统。

重点监控：

- 极低生命 + 极高爆发。
- 吸血 + 攻速 + 低血伤害叠成不死输出。
- 技能急速堆到无限循环。
- DOT 增幅 + 效果强度 + 减抗叠乘爆炸。
- 护盾强度 + 受治愈增幅 + 净化导致打不完。
- 单个部位或单个词条贡献过高。

计算方向：

```text
输出预算 / 生存预算 / 节奏预算 / 机制预算
单项超过阈值并伴随异常战斗结果 => 标记风险
```

高输出不是错误。真正的问题是代价不真实。

### 5.4 Formation Speed：成型速度

问题：玩家刷多久能得到可玩的提升。

测试方式：

```text
模拟 10、20、50、100、200 件掉落
每个阶段自动配装
观察目标 build 分数和战斗表现增长
```

理想结果：

- 10-20 件装备应有可感提升。
- 30-50 件能出现初步 build。
- 80-120 件进入优化和追求高等级词条。
- 不应该刷 100 件还没有任何能玩的构筑。

异常：

- 关键流派词条太稀有。
- 关键词条被锁在太少部位。
- 词条等级太吃高稀有度。
- 固定属性太强，随机词条不重要。
- 随机词条太强，固定属性不重要。

## 6. 装备水表和战斗验证

静态评分之后，必须进入战斗验证。

使用更丰富的水表：

- 无装备基准队。
- 固定 10 点加点队。
- 固定 20 点加点队。
- 固定 30 点加点队。
- 装备后标准队。
- 装备后流派强化队。

战斗验证看：

- 装备是否真的改变曲线。
- 装备是否只提高胜率但没有增强幻想。
- 装备是否导致极端克制。
- 装备是否让某些职业突然超标或彻底失效。

战斗后的归因必须使用多项对比：

```text
发生：问题事件怎么出现
扩大：什么词条/部位/数值让它变大
解决：对手为什么解决不了
反证：改动一个装备规则后曲线如何变化
```

## 7. 调整规则

每轮调整只能动装备系统。

可调整项：

- 修改部位固定属性。
- 修改部位词条池。
- 修改词条数值。
- 修改词条等级曲线。
- 修改稀有度概率。
- 修改词条稀有度门槛。
- 增加同类词条互斥。
- 增加同件装备乘区上限。
- 将过强词条从常见部位移到少数部位。
- 给弱流派增加更多成型入口。

不可调整项：

- 不改职业基础数值。
- 不改技能数值。
- 不改战斗引擎。
- 不改水表队伍来让装备看起来更好。

## 8. 循环预算

最多执行 5 轮。

每轮都要记录：

- 本轮装备规则版本。
- 掉落模拟参数。
- 四个评价函数结果。
- 装备水表结果。
- 发现的问题。
- 归因。
- 改动内容。
- 本轮是否优于上一轮。
- 是否保存为阶段性最佳。

如果第 3 轮已经明显好，可以保存为 `best-so-far`，然后用剩余轮数探索替代方案。

如果连续两轮没有提升，停止堆数值，改做归因或流程检查。

## 9. 每轮给用户的汇报格式

每轮结束必须在聊天里汇报，不只写 report。

格式：

```text
第 X/5 轮完成。

本轮规则变化：
- ...

四个评价：
- 多样性：...
- 流派增强：...
- 极端风险：...
- 成型速度：...

战斗水表观察：
- ...

我认为最大问题是：
- ...

下一轮只改：
- ...
```

如果写 coop report，也要把 report 的主要内容直接在聊天里讲给用户。

## 10. 最终输出

5 轮结束或提前收束后，输出：

- 最佳装备规则版本。
- 装备 affix registry 草案。
- 掉落生成器参数。
- 自动配装偏好函数。
- 四个评价函数的最终结果。
- 战斗水表摘要。
- 已知风险。
- 下一阶段建议。
