# 装备 combat proxy 变体对比

状态：第 5/5 轮验证摘要。

约束：没有修改角色属性、技能、战斗引擎或水表队伍。所有结果都来自克隆队伍 spec 上的装备代理映射。

## 结果

| 版本 | 静态定位 | 平均胜率变化 | 胜转负数量 | 观察 |
| --- | --- | ---: | ---: | --- |
| v1 基线 | 原始规则 | -2.0% | 8 | 没有形成稳定战斗收益 |
| v2 更快成型 | 提高优质/稀有和流派词条 | +1.1% | 8 | 有轻微收益，但仍有回退 |
| v3 部位价值平衡 | 削弱挂饰、增强防御/功能位 | 0.0% | 7 | 回退最少，但几乎没有整体收益 |
| v4 极端风险收束 | 静态最佳 | -3.1% | 9 | 静态分高，但 combat proxy 不稳 |
| v5 固定底座回补 | 低风险候选 | +2.1% | 8 | 当前 combat proxy 中最适合作为下一阶段候选 |

## 结论

- 静态评价和 combat proxy 并不完全一致，这是好发现。
- v4 静态分最高，但进入战斗代理后不稳。
- v5 静态不是最高，但战斗代理收益最正、风险相对可控，因此作为下一阶段候选更合适。
- 当前仍有 7-9 个胜转负，说明装备映射不能直接当最终公式。

## 下一步

下一阶段应该做正式装备公式接入，而不是继续调代理：

- 把 `lifeSteal`、低血、标记、隐身、DOT、护盾等机制从“代理成 power/hp/haste”升级为战斗可识别的装备 modifier。
- 水表验证时保留静态四指标，同时加入真实战斗信号指标。
- 在 UI 或报告里区分“静态 build 好”和“战斗曲线真的好”。
