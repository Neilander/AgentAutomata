# 刷装模拟器 UI 方案

## 玩家意图

玩家进入页面是为了验证一个休闲刷装循环：用当前角色挑战副本，拿到装备，给合适的人穿上，再挑战更高等级。

## 核心对象

核心对象是“当前小队 vs 当前副本”的成长循环，而不是背包清单本身。

## 信息优先级

- P0：当前小队战力、当前副本、开刷按钮、战斗过程。
- P1：敌方预览、最近掉落、当前角色装备。
- P2：装备详细词条、装备评分变化、角色技能概览。
- P3：背包完整内容、历史掉落。
- P4：随机种子、精细调参、原始战斗日志，当前版本暂不露出。

## 布局

- 顶部：整体进度，小队战力，最高通关。
- 中区左：5 档副本选择和本次敌人预览。
- 中区中：实时战斗窗口，保持最大注意力。
- 中区右：最近收益，强调掉落反馈。
- 下区左：6 个角色，控制上阵/下阵。
- 下区中：当前角色装备位。
- 下区右：固定格背包和选中装备详情。

## 控件选择

- 副本：少量互斥选项，用卡片按钮。
- 角色：6 个固定对象，用紧凑角色卡。
- 背包：装备需要扫稀有度和部位，用固定格子，不用长列表。
- 详情：只显示当前选中的角色或装备，避免重复文字。

## 验收重点

- 玩家 2 秒内能看到下一步是“开刷一次”。
- 战斗窗口不会被背包或日志挤压。
- 装备是格子和图标，不是文字列表。
- 赢了以后有明显掉落反馈。
- 选角色、点装备、装备到当前角色这一链路空间相邻。
