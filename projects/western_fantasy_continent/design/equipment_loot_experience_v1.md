# 装备刷取体验 v1 草案

状态：设计草案，未实装。

目标：先把“刷装备为什么爽、装备怎么限制极端 build、第一版做到什么规模”说清楚。本文只定义系统形状，不直接写掉落表和数值公式。

## 1. 设计目标

装备系统要服务三个体验：

1. **刷到新装备有即时反馈。**
   玩家看到颜色、基础属性和词条数量，就能快速判断“这件可能有用”。

2. **刷到好词条有追求空间。**
   同一稀有度内，词条类型和词条等级决定装备质量。玩家不是只看颜色。

3. **装备不能让角色畸形堆单项。**
   部位限制词条池，避免全身都堆攻击、攻速、生命或同一种流派增伤。玩家可以专精，但不能把一个维度堆到失控。

参考方向：类似 Dark and Darker 的“装备基础属性 + 随机词条 / 附魔 + 装备部位限制词条池”，但本项目要服务自动战斗、4人配队和 combo 生态，不照搬动作游戏数值。

## 2. 装备栏位

第一版角色装备位：

| 栏位 | 定位 |
| --- | --- |
| 头 | 认知、法术、节奏、抗效果 |
| 胸 | 主生存位 |
| 护手 | 主输出手感位 |
| 腿 | 次生存位、站场稳定性 |
| 鞋 | 行动节奏、先手、灵活性 |
| 挂饰 | 流派、特殊机制、稀有词条入口 |
| 左手 | 副武器、防具、触发器、辅助功能 |
| 右手 | 主武器、主要攻击/法术基础 |

武器规则：

- 普通角色可以装备左手 + 右手。
- 双手武器占用左手和右手两个栏位。
- 盾牌、法器、副手匕首、书卷等走左手。
- 大剑、大锤、长弓、法杖等可以设计成双手武器。

第一版可以先只做：

- 单手武器
- 副手盾 / 法器
- 双手武器

不要一开始扩太多武器分类。

## 3. 稀有度和词条数量

数据结构可以支持很高上限，但前期验证不要开放到神话九词条。

第一版推荐做到“史诗 4 词条”左右：

| 稀有度 | 颜色定位 | 词条数 | 第一版状态 |
| --- | --- | ---: | --- |
| 粗糙 | 灰 | 0 | 可做掉落下限 |
| 普通 | 白 | 1 | 可做 |
| 优质 | 绿 | 2 | 可做 |
| 稀有 | 蓝 | 3 | 可做 |
| 史诗 | 紫 | 4 | 第一版上限 |
| 传说 | 橙 | 5 | 预留 |
| 神话 | 红 / 彩 | 6-7 | 预留 |
| 神器 | 特殊 | 8-9 | 预留 |

后期可以开放：

- 神话不一定固定 9 条，也可以是 6-7 条高等级词条 + 机制词条。
- 神器的价值不一定靠纯数值，而是靠特殊机制和极高词条等级上限。

## 4. 基础属性与随机词条

每件装备由两部分组成：

```text
装备 = 部位基础属性 + 随机词条
```

### 基础属性

基础属性由装备位决定，代表这个部位天然提供什么价值。

基础属性随：

- 装备等级
- 装备稀有度
- 装备类型

共同变化。

例如：

- 胸甲天然有生命 / 防御。
- 鞋天然有速度 / 行动节奏。
- 右手武器天然有攻击或法强。
- 盾牌天然有防御 / 护盾相关基础。

### 随机词条

随机词条决定 build 方向和刷装追求。

词条有两个维度：

- 词条类型：武力、敏捷、攻击、攻速、技能急速、燃烧增幅等。
- 词条等级：I、II、III、IV、V 等。

同一件装备原则上不重复出现完全相同词条。

如果某个属性已经是该装备的核心基础属性，第一版可以避免再滚出同名随机词条，减少“基础攻击 + 随机攻击 + 武力全堆”的极端情况。

## 5. 词条等级成长

词条等级可以故意做成陡峭曲线，让高等级词条非常有价值。

候选曲线：

| 词条等级 | 数值预算 |
| --- | ---: |
| I | 1 |
| II | 2 |
| III | 4 |
| IV | 7 |
| V | 12 |

这不是最终平衡，只是体验方向：

- 低等级词条是添头。
- III 级开始明显有感。
- IV/V 级会让玩家兴奋。
- 高稀有度装备的追求不只是多词条，也包括高等级词条。

后期可以扩展：

| 词条等级 | 数值预算 |
| --- | ---: |
| VI | 18 |
| VII | 26 |
| VIII | 37 |
| IX | 52 |

但第一版不建议开放到这里。先保留数据结构。

## 6. 部位词条池

部位词条池是装备系统的平衡阀门。

它的目的不是拟真，而是防止玩家全身堆同一类属性，把角色推成极端 build。

### 第一版部位限制

| 部位 | 基础属性倾向 | 允许随机词条 | 限制 / 不给 |
| --- | --- | --- | --- |
| 头 | 技能节奏、法术、抗效果 | 奥术、节律、技能急速、法强、效果抗性、暴击率 | 不给大量生命；不做主防御位 |
| 胸 | 主生存 | 坚韧、韧性、生命、防御、受治愈增幅、效果抗性 | 不给攻击、法强、攻速、暴击伤害 |
| 护手 | 主输出手感 | 武力、敏捷、攻击、攻速、暴击率、暴击伤害、技能伤害 | 不给大量生命；不给主防御 |
| 腿 | 次生存、稳定 | 坚韧、韧性、生命、防御、效果抗性、少量技能急速 | 不给高攻击、高法强、高暴击 |
| 鞋 | 行动与节奏 | 敏捷、攻速、技能急速、效果抗性、先手、闪避类 | 不给大量生命、防御、爆发增伤 |
| 挂饰 | 流派入口 | 大属性、技能急速、效果强度、DOT增幅、治疗/护盾强度、特殊触发 | 基础防御较弱；强机制要有稀有度门槛 |
| 右手 | 主输出基础 | 攻击、法强、武力、奥术、暴击、技能伤害、流派伤害 | 不给主生存词条 |
| 左手 | 副功能 | 防御、护盾强度、技能急速、效果强度、格挡/保护、辅助触发 | 输出上限低于右手 |
| 双手 | 高基础输出 / 高机制 | 攻击、法强、武力、奥术、暴击伤害、技能伤害、专属机制 | 占用双手，不能再拿盾或副手功能 |

### 明确限制示例

- 胸和腿原则上不出直接伤害词条。
- 护手和右手是主要输出词条来源。
- 鞋提供节奏，不提供爆炸伤害。
- 挂饰可以出奇怪东西，但基础数值不能同时也很高。
- 双手武器可以强，但它的代价是失去左手副功能。

## 7. 词条全集和分层

装备第一版必须先承认一件事：词条不是只有六个大属性。真正影响刷装体验的是“大属性 + 一级小词条 + 二级专精词条 + 三级流派机制词条”的组合。

本节把目前项目里已经整理过或出现过的词条合并到一张设计口径里：

- v2 正式属性来源：`attribute_system_v2_candidate.md`、`attribute-build-route-simulation-v2.md`、`attribute-team-route-simulation-v2.md`、`game_data/analyze-attribute-build-routes-v2.js`。
- 旧装备/大地图实装字段来源：`game_data/stat-schema.js`、`game_data/equipment.js`、`world_map_demo/world-map.js`。
- 关键词机制来源：`game_data/keyword-mechanics.js`。

旧实装字段不能直接当新设计照搬；它们要迁移成 v2 语言。例如 `magicResist` 以后优先并入“效果抗性/韧性”体系，`moveSpeed` 以后优先并入“先手/机动”，`accuracy/evasion` 暂时作为预留命中闪避词条，不作为 v1 核心。

### 7.1 大属性词条

大属性是复合成长，适合中高稀有度装备。

| 大属性 | 主收益 | 副产物 | 主要服务职业 |
| --- | --- | --- | --- |
| 武力 | 攻击 | 生命 | 战士、游侠、狂战士、刺客副收益 |
| 坚韧 | 生命 | 防御 | 战士、骑士、前排狂战 |
| 敏捷 | 攻速 | 效果抗性 | 狂战士、刺客、游侠 |
| 奥术 | 法强 | 技能急速 | 法师、牧师、术士、炼金师、吟游副收益 |
| 节律 | 技能急速 | 效果强度 | 法师、术士、吟游诗人、炼金师 |
| 韧性 | 防御 | 效果抗性 | 骑士、牧师、抗 DOT/抗控制角色 |

### 7.2 一级小词条：基础战斗公式

一级词条是玩家最容易理解、也最适合大量掉落的基础数值。它们可以出在多个部位，但要限制直接输出和直接生存的堆叠位置。

| 词条 | 定位 | 主要部位 | 禁区 / 限制 |
| --- | --- | --- | --- |
| 攻击 | 物理普攻和物理技能基础 | 右手、双手、护手 | 胸、腿不出 |
| 法强 | 法术、治疗、护盾和部分效果基础 | 右手、双手、头、左手 | 胸、腿不出；鞋只预留特殊鞋 |
| 生命 | 最大生命 | 胸、腿、少量头/左手 | 右手、双手不出大量生命 |
| 防御 | 单发伤害承伤 | 胸、腿、左手 | 右手、护手不做主防御 |
| 攻速 | 普攻频率、on-hit、吸血触发 | 护手、鞋、右手 | 胸、腿不出 |
| 技能急速 | 缩短技能冷却 | 头、鞋、左手、挂饰 | 护手少量；胸腿只低权重 |
| 效果抗性 | 抗 DOT、控制、诅咒、减速 | 头、胸、腿、鞋、左手 | 不应堆到完全免疫 |

### 7.3 二级小词条：专精成长

二级词条比一级更窄，但更能形成 build。它们不能只放挂饰，必须分布到相关装备位。

| 词条 | 定位 | 主要部位 | 备注 |
| --- | --- | --- | --- |
| 效果强度 | 提高 DOT、控制、诅咒等效果 | 头、左手、挂饰、法杖/双手 | 术士、炼金、控制流核心 |
| 治疗强度 | 提高主动治疗量 | 头、左手、挂饰 | 牧师主入口，诗人少量 |
| 护盾强度 | 提高主动护盾量 | 胸、左手、挂饰 | 骑士/牧师；要防止护盾拖死节奏 |
| 受治愈增幅 | 自己受到治疗/护盾更多 | 胸、腿、左手 | 狂战、骑士、前排容器 |
| 暴击率 | 稳定爆发机会 | 头、护手、右手 | 游侠、刺客、战士 |
| 暴击伤害 | 高风险爆发收益 | 护手、右手、双手 | 不进胸、腿、鞋 |
| 吸血 | 造成伤害后回血 | 护手、右手、双手 | 少量挂饰可出；不要泛用化 |
| 真伤 | 绕过防御/护盾的破局 | 右手、双手、少量挂饰 | 高稀有预留，v1 不建议大量开放 |
| DOT 增幅 | 提高持续伤害 | 头、右手、双手、挂饰 | 燃烧/剧毒/诅咒共用乘区 |
| 控制强度 | 控制时长或控制效果 | 头、鞋、左手、挂饰 | 控制不能完全锁死 |
| 破盾 | 打护盾队 | 护手、右手、双手 | 战士、游侠、刺客可用 |
| 破甲 | 打高防御前排 | 护手、右手、双手 | 和破盾可以拆开，避免一条太全能 |
| 先手 | 开局先行动/缩短首轮冷却 | 鞋、头、挂饰 | 刺客、游侠、控制流 |
| 格挡 | 减少正面单发伤害或触发反击 | 左手、胸、腿 | 盾牌核心，不应在右手出现 |
| 闪避 | 小概率规避单发伤害 | 鞋、头、挂饰 | 预留；不能同时高攻速高闪避全身堆 |
| 命中 | 克制闪避/隐身/机动 | 头、右手、护手 | 预留；等闪避体系成型再开放 |

### 7.4 三级词条：流派和机制

三级词条直接指向 combo、职业幻想或特定反制。它们不是全部塞挂饰，而是按“谁在制造这个体验”分布到对应部位。

| 词条 | 服务流派 | 主要部位 | 说明 |
| --- | --- | --- | --- |
| 物理伤害增幅 | 战士、游侠、刺客、狂战 | 右手、双手、护手 | 比攻击更窄，数值可以高一点 |
| 火焰伤害 | 火法直伤/燃烧基础 | 右手、双手、头 | 对应旧 `fireDamage` |
| 火焰增幅 | 火法爆发/燃烧爆发 | 头、右手、双手、挂饰 | 对应旧 `fireDamageAmp` |
| 燃烧持续 | 燃烧铺场 | 头、挂饰、少量胸 | 对应旧 `fireDurationAmp`，胸只给防御型燃烧抗压装 |
| 剧毒伤害 | 毒队持续输出 | 右手、双手、护手、挂饰 | 对应旧 `poisonDamage` |
| 剧毒增幅 | 毒层兑现 | 头、右手、双手、挂饰 | 对应旧 `poisonDamageAmp` |
| 剧毒持续 | 毒层稳定 | 头、腿、挂饰 | 对应旧 `poisonDurationAmp` |
| 闪电伤害 | 连锁/高频触发 | 右手、双手、护手 | 对应旧 `lightningDamage` |
| 闪电连锁 | 连锁清场 | 护手、右手、挂饰 | 对应旧 `lightningChainChance` |
| 冰霜伤害 | 控制法/减速 | 头、左手、右手、双手 | 对应旧 `iceDamage` |
| 冰霜减速 | 减速和控制窗口 | 头、鞋、左手、挂饰 | 对应旧 `iceSlowAmp` |
| 神圣增幅 | 治疗、庇护、净化 | 头、左手、挂饰 | 对应旧 `holyDamageAmp`，以后可改名神圣强度 |
| 暗影增幅 | 暗影刺客/术士 | 右手、双手、挂饰、头 | 对应旧 `shadowDamageAmp` |
| 奥术增幅 | 法师/炼金/术士技能 | 头、右手、双手、左手 | 对应旧 `arcaneDamageAmp` |
| 范围伤害增幅 | AOE 法师、炼金 | 双手、头、挂饰 | 对应旧 `areaDamageAmp` |
| 单体伤害增幅 | 点杀、刺客、游侠 | 右手、护手、头 | 对应旧 `singleTargetDamageAmp` |
| 标记强度 | 游侠/刺客点杀 | 头、护手、右手、挂饰 | 标记生产、标记伤害、标记转火可以后续拆 |
| 隐身持续 | 暗影刺客 | 鞋、头、挂饰 | 不只在挂饰；鞋给切入稳定 |
| 处决伤害 | 低血收割 | 右手、双手、护手、挂饰 | 高风险，避免过早秒前排 |
| 低血伤害 | 狂战、濒死反打 | 护手、右手、双手、挂饰 | 不能和生存词条在同部位过度叠 |
| 低血受治愈增幅 | 狂战、前排容器 | 胸、腿、左手、挂饰 | 让低血队需要输出位+生存位共同成型 |
| 反击伤害 | 铁壁、盾反 | 左手、胸、护手、挂饰 | 盾牌/前排优先，不放右手纯输出池 |
| 嘲讽减伤 | 骑士前排 | 胸、腿、左手 | 防御型流派词条 |
| 护盾转伤害 | 护盾队兑现 | 左手、胸、挂饰 | 高风险，数值要保守 |
| 净化效率 | 牧师、反 DOT | 头、左手、挂饰 | 克制毒/火/控制 |
| 光环强度 | 诗人团队增益 | 头、左手、挂饰、鞋 | 不是直接个人输出 |
| 召唤/随从强度 | 预留召唤玩法 | 挂饰、左手、头 | v1 可先不开放 |
| 易伤增幅 | 团队集火 | 头、右手、挂饰 | 对应旧 `vulnerabilityAmp`，谨慎开放 |
| 减抗/削抗 | 效果流破抗 | 左手、头、挂饰 | 对应旧 `resistShred`，必须防止 DOT 失控 |

## 8. 部位词条池初版

部位词条池不是“哪个部位塞几个好听的词”，而是生成器之后真正抽词条的限制表。每个部位都要有一级、二级、三级入口。

### 8.1 部位到词条池

| 部位 | 大属性池 | 一级小词条 | 二级小词条 | 三级/流派词条 |
| --- | --- | --- | --- | --- |
| 头 | 奥术、节律、韧性 | 法强、技能急速、效果抗性 | 效果强度、治疗强度、控制强度、暴击率、命中 | 火焰增幅、燃烧持续、剧毒增幅、剧毒持续、冰霜减速、神圣增幅、奥术增幅、标记强度、隐身持续、净化效率、光环强度 |
| 胸 | 坚韧、韧性 | 生命、防御、效果抗性 | 护盾强度、受治愈增幅、格挡 | 燃烧持续、低血受治愈增幅、反击伤害、嘲讽减伤、护盾转伤害、DOT 抗压 |
| 护手 | 武力、敏捷 | 攻击、攻速 | 暴击率、暴击伤害、吸血、破盾、破甲、命中 | 物理伤害增幅、闪电连锁、标记强度、单体伤害增幅、处决伤害、低血伤害、反击伤害 |
| 腿 | 坚韧、韧性、少量敏捷 | 生命、防御、效果抗性 | 受治愈增幅、格挡、少量技能急速 | 剧毒持续、低血受治愈增幅、嘲讽减伤、抗控制、站场稳定 |

### 8.2 明确禁区

这些禁区是为了保护 build 形状：

- 胸、腿不出直接攻击、法强、暴击伤害、单体伤害增幅。
- 鞋不出大量生命、防御、暴击伤害、处决伤害。
- 右手、双手不出大量生命、受治愈增幅、主防御。
- 护手不出大量生命、防御、治疗强度、护盾强度。
- 挂饰可以出很多机制，但基础属性预算必须低，不能同时是高基础数值和强机制。
- 左手可以出支援、防御、法器异常，但不要比右手更适合做纯输出。
- 真伤、减抗/削抗、易伤增幅、护盾转伤害属于高风险词条，v1 只作为高稀有预留或测试词条。

## 9. 职业覆盖检查

按照 `analyze-attribute-build-routes-v2.js` 和 v2 报告里的职业主副属性，装备词条池要保证每个职业至少有：

- 一个主属性可刷部位。
- 一个副属性可刷部位。
- 一个窄专精属性可刷部位。
- 一个不走主副属性也有意义的队伍/流派入口。

| 职业 | 主副属性 | 主要刷装部位 | 可用专精词条 | 设计检查 |
| --- | --- | --- | --- | --- |
| 战士 | 武力 / 坚韧 | 右手、护手、胸、腿、双手 | 破盾、破甲、反击伤害、受治愈增幅 | 能走输出或半前排，不被迫全武力 |
| 狂战士 | 敏捷 / 武力 | 护手、鞋、右手、双手、胸 | 攻速、吸血、低血伤害、低血受治愈增幅 | 低血流需要输出位和生存位共同成型 |
| 骑士 | 坚韧 / 韧性 | 胸、腿、左手、挂饰 | 护盾强度、受治愈增幅、效果抗性、反击 | 防御和守护都能刷，不只堆坚韧 |
| 游侠 | 武力 / 敏捷 | 护手、鞋、右手、双手、头 | 标记增幅、攻速、暴击率、先手 | 物理输出和高频触发都有装备入口 |
| 法师 | 奥术 / 节律 | 头、右手、双手、挂饰、鞋 | 法强、技能急速、元素/DOT、效果强度 | AOE、DOT、技能循环都能被装备支持 |
| 牧师 | 奥术 / 韧性 | 头、胸、左手、挂饰、腿 | 治疗强度、护盾强度、效果抗性、净化 | 后排治疗和抗压牧师都有装备入口 |
| 术士 | 奥术 / 节律 | 头、挂饰、右手、双手、左手 | DOT 增幅、效果强度、献祭承载、技能急速 | 异常输出和循环必须拆在不同部位 |
| 吟游诗人 | 节律 / 奥术 | 头、鞋、挂饰、左手 | 光环强度、团队增益、技能急速、控制强度 | 团队节奏和功能强度都有入口 |
| 刺客 | 敏捷 / 武力 | 护手、鞋、右手、挂饰、头 | 隐身持续、标记伤害、暴击率、先手 | 切入频率和击杀质量都能做，但不能全身堆暴伤 |
| 炼金师 | 节律 / 奥术 | 头、挂饰、左手、右手、双手 | 效果强度、DOT 增幅、控制强度、技能急速 | 异常循环和爆发兑现都有入口 |

### 分配后的风险

- 狂战和刺客都会想要敏捷、攻速、吸血、先手，必须用部位分散限制。
- 法师、术士、炼金都会吃奥术/节律、效果强度、DOT 增幅，DOT 增幅不能在太多部位出现。
- 牧师和骑士都会吃韧性、护盾强度、效果抗性，护盾强度要有上限或递减。
- 战士、游侠、刺客都会吃武力和暴击/标记类输出，暴击伤害要集中在护手/武器，不要进鞋/胸/腿。

## 11. 强度预算原则

一件装备的强度来自：

```text
基础属性预算
+ 词条数量预算
+ 词条等级预算
+ 词条类型稀有度预算
+ 特殊机制预算
```

不要让所有预算同时拉满。

例如：

- 普通高等级装备：基础属性不错，但词条少。
- 史诗低等级装备：词条多，但基础属性不一定碾压。
- 神话装备：可以有高等级词条，但特殊机制需要牺牲部分基础预算。
- 神器装备：更像 build 改写器，不只是大数字。

## 12. 刷装体验结构

一次掉落的判断顺序应该是：

1. 看部位：是不是我缺的栏位。
2. 看稀有度：词条数量是否值得看。
3. 看基础属性：这个部位本身底子好不好。
4. 看词条类型：是否适配角色 / 队伍 / 流派。
5. 看词条等级：有没有高等级词条。
6. 看组合：多个词条是否形成 combo。

玩家想刷的不是“战力数字更高”，而是：

- 给狂战刷低血 + 攻速 + 受治愈增幅。
- 给刺客刷敏捷 + 隐身 + 标记伤害。
- 给火法刷奥术 + 技能急速 + 燃烧增幅。
- 给骑士刷坚韧 + 韧性 + 护盾/反击。
- 给牧师刷奥术/节律 + 治疗/护盾强度。

## 13. 第一版实现范围建议

第一版只做可验证闭环：

- 8 个装备位：头、胸、护手、腿、鞋、挂饰、左手、右手。
- 支持双手武器占双手。
- 稀有度做到史诗。
- 词条数量最多 4 条。
- 词条等级做到 V。
- 每个部位一套基础属性池。
- 每个部位一套随机词条池。
- 先做 20-30 个常用词条，不做完整词条宇宙。
- 先接入 4v4 战斗验证，不急着做完整掉落经济。

第一版不做：

- 神话 / 神器正式掉落。
- 9 词条装备。
- 套装系统。
- 宝石镶嵌。
- 复杂耐久和交易。
- 完整装备评分算法。

## 14. 下一步需要细化

下一轮设计建议按这个顺序：

1. 确定第一版装备位是否就是 8 格 + 双手占用。
2. 给每个部位列基础属性池。
3. 给每个部位列随机词条池。
4. 把词条分成通用、大属性、小属性、流派四类。
5. 设计第一批 20-30 个词条。
6. 定义装备生成伪代码。
7. 做一个刷装预览页或命令行生成器。
8. 把 3-5 套角色 build 放进 4v4 战斗验证。

当前最重要的设计判断：

> 装备系统的上限来自“部位限制下的词条组合”，而不是全身无脑堆同一种最强属性。
